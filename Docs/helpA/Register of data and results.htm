Register of data and result

 

Data include regulation and assets, security properties, threat, attack, attacker.

Each of these data groups may contain no more than 10 items in one data model and participate in the analysis if it is a non-empty set of characters. The method establishes the relationship between the components. The data register contains the values of risk factors and the calculation of the risk itself.

Data and calculation results are saved in an XML file. The file includes: name, and description referring to the basic data of the instrument in the context of threat estimation, dictionary data:

regulation1, …, regulation10,

impact1, …, impact10,

assets1, …, assets10,

security_properties1, …, security_properties10,

threat1, …, threat10,

attack1, …, attack10,

attacker1, …, attacker10,

and relational data:

…

regulation10_assets1, …, regulation10_assets10,

assets1_security_properties1, …, assets1_security_properties10,

…

…

…
