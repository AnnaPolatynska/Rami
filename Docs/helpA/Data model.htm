Data model

The data model includes a set of regulations that has influence on the identification of assets. Assets come directly from the regulations, they can be repeated if the regulations refer to the same assets. It is important to identify assets for each legal regulation that participates in the model. It is not possible to refer directly to such values as public health or financial losses (although they could occur in the case of falsifying the reading of a measuring instrument) they should be referred to other values such as the security properties of a measuring instrument. For each measuring instrument using software, you can choose between: unavailability of software or availability (measurement identification). Other possible security properties include integrity (measurement values) or software authenticity. There is a connection between the asset and the security property, which facilitates the identification of assets with a limited set of protection property elements - in the case of measuring instrument using the software, four security properties are sufficient to describe all cases.

Another aspect of the model are threats that have an adverse effect on assets. Example of threat: "Local administrator invalidates integrity or authenticity of the metrological software".  The example refers to an asset (metrological software) and shows the relationship with two security properties: integrity or authenticity and indicates the attacker (local administrator) and the activities introducing threat. It can be said that the occurrence of a threat requires three types of components: an attacker, assets and security properties. With defined protection properties, the model should also have two other groups of factors, which at the same time determine the consistency of the model: the threat is related with the attack, the attack is carried out by the attacker who is interested in violating the security properties assigned to some assets.

In the model, the threat implement attack. Example of attack: "An attacker fakes an authorization key on a USB stick thus gaining access to the operating system". This means the implementation of the following threat: "Local administrator invalidates integrity or authenticity of the metrological software".

There is no attack in the model if it doesn implement at least one threat.

Data model can be represented as:

ü  the regulation has 1 or more assets,

ü  the assets have 1 or more security properties,

ü  the attack has 1 or more assets,

ü  the attack has 1 or more threats,

ü  the attack has 1 or more performers,

ü  the attack carries one risk.
