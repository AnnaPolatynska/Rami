Risk analysis

The analysis starts with the creation of a new document with the “New” option.

In the "Instrument" tab, basic data are entered: the name and description of the measuring instrument in the context of risk estimation.

Then, in the ”Dictionaries” tab, fill the data for regulation and assets in the “Regulations / Assets” tab, protection properties, threats, attacks and attackers in the other tabs. The next step is to establish the relationship between the data in the Relationships tab, which concern:

ü  Regulations with assets,

ü  Assets with security properties,

ü  Attacks with assets,

ü  Attacks with threats,

ü  Attacks with attackers,

ü  Attacks with risk.

Content of all relations except the last one is defined in the lists.

For example, the relation Regulations - Assets for the following regulation: ”When a measuring instrument has associated software which provides other functions besides the measuring function, the software that is critical for the metrological characteristics shall be identifiable and shall not be inadmissibly influenced by the associated software“, has three assets:

 

ü  identification of the software,

ü  inadmissible influence on the software,

ü  Important metrological parameters.

After establishing the relationship, you can check the consistency of the data (“Check” option) and correct the data model if Rami detects any errors.

Atak – risk relation requires determining the value of risk factors on appropriate sliders. During these activities, the user is informed about the value of corresponding parameter and the result of the risk calculation.

The entered data can be saved in the XML file with “Save” or “Save As” option.

If you don’t want to save the changes you must use “Close” option and give the answer “no” to the question "Do you want to save changes?". This will close the document without saving it.
